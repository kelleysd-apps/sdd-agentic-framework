# prd-specialist Context

This directory stores working context and state for the prd-specialist agent.

## Purpose

Context files capture the current working state of PRD creation sessions, including:
- Active PRD being worked on
- User interactions and clarifications
- Pending questions and decisions
- Session progress tracking

## Files

Context files are typically created during active PRD sessions and may include:
- Current PRD draft state
- User answers to discovery questions
- Temporary notes and observations
- Session checkpoints

## Usage

The prd-specialist agent references this context to:
- Resume interrupted PRD sessions
- Maintain continuity across conversations
- Track progress through PRD sections
- Remember user preferences and constraints
